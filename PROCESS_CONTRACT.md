# Babel Microgame Process Contract

Microgame: 坠物预警

This file is generated by the Babel s control plane. It exists so every new or existing microgame repository immediately inherits the same production workflow.

## Authority

- /home/openclaw/babel-runtime is the process authority.
- /home/openclaw/claudecode-manager is the Codex manager lane.
- /home/openclaw/Babel is the m/Babel implementation lane.
- This repository must not fork or reimplement those controls.

## Required Flow

1. s receives or normalizes product input.
2. Codex manager creates or continues a per-game queue.
3. ClaudeCode worker implements only the current packet.
4. Worker runs babel-ops worker finish.
5. Codex manager reviews with microgame_worker_review_handoff.sh.
6. Both finish and review leave GitHub audit issues through babel-ops manager audit.

Those audit issues must include commit trace links generated by s: the issue body and closing comment both point to the current HEAD GitHub commit.

## Quality Gates

- Direction Lock is mandatory and higher priority than worker preference.
- Direction Check is mandatory in every worker report.
- Scope, budget, acceptance, test command, and scripted playthrough constraints are blocking gates.
- Only one worker may actively modify a game repo unless the manager has explicit non-overlapping scopes.
- Rework must be recorded as rework, not silently fixed by another worker.

## Prohibited Shortcuts

- Do not create per-repo issue-management scripts.
- Do not launch extra Codex or ClaudeCode sessions from this repo.
- Do not consume s or m issue spaces directly.
- Do not treat local .codex-runtime registry edits as a substitute for GitHub audit issues.
- Do not expand gameplay beyond the Direction Lock to compensate for unclear packets.

## Fast Migration Rule

Whenever s changes the production workflow, run:

~~~sh
/home/openclaw/babel-runtime/.codex-runtime/bin/babel-ops microgame sync-contract --game-workdir "$PWD" --commit --push
~~~

For all existing repos, s may run:

~~~sh
/home/openclaw/babel-runtime/.codex-runtime/bin/babel-ops microgame sync-contract --all --commit --push --skip-dirty
~~~
