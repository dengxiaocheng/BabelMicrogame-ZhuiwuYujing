# MINI_GDD: 坠物预警

## Scope

- runtime: web（纯 HTML/CSS/JS，无框架依赖）
- duration: 20min（8 波 × ~2.5min）
- project_line: 坠物预警
- single_core_loop: 观察前兆 -> 发出预警或继续工作 -> 工人反应 -> 事故/产量结算 -> 下一波

```
START
  │
  ▼

     ▼                         │
```

## State Init

| State | Init | Min | Max | 说明 |
|-------|------|-----|-----|------|
| warning_tokens | 3 | 0 | 3 | 每波重置 |
| quota | 100 | 0 | 100 | 降到 0 触发产量结局 |
| injury_risk | 0 | 0 | 100 | 升到 100 触发工伤结局 |
| false_alarm | 0 | 0 | 100 | 升到 100 触发信任结局 |
| wave | 1 | 1 | 8 | 当前波次 |

## Wave Structure（每波 3 阶段）

1. **前兆期（45s）**：显示视觉前兆，玩家可拖拽预警旗到 Zone。warning_tokens 每波初重置为 3。
2. **决策锁定期（15s）**：停止接收拖拽，播放工人反应动画。
3. **结算期（5s）**：根据 Zone 是否被预警 + 是否真有坠物，更新 state，检查结局。

## Work Zones

| Zone | 位置 | 工人数 | quota贡献/波 |
|------|------|--------|-------------|
| A | 左侧塔底 | 3 | 4 |
| B | 中间塔底 | 4 | 5 |
| C | 右侧塔底 | 3 | 4 |

- 每波最多 2 个 Zone 同时有坠物危险（wave 5+ 允许 3 个）
- 预警 Zone 的工人停工（不贡献 quota，但安全）
- 未预警且真有坠物的 Zone：工人受伤，injury_risk +25

## Endings

| 结局 | 条件 | 文案方向 |
|------|------|---------|
| 好结局 | wave=8 且 quota>=60 且 injury_risk<50 | "安全与效率的平衡" |
| 普通结局 | wave=8 且不满足好结局 | "勉强过关" |
| 工伤结局 | injury_risk>=100 | "工人的代价" |
| 信任结局 | false_alarm>=100 | "狼来了" |
| 产量结局 | quota<=0 | "效率的代价" |

## Constraints

- 总体规模目标控制在 5000 行以内
- 单个 worker 任务必须服从 packet budget
- 如需扩线，交回 manager 重新拆
- 不加多余菜单和后台页
