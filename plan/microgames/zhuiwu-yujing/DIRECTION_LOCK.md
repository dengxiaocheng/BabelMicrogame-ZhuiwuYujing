# Direction Lock: 坠物预警

## One Sentence
玩家识别塔上坠物前兆，决定何时让工人躲避而不耽误产量。

## Core Loop
1. 执行核心循环：观察前兆 -> 发出预警或继续工作 -> 工人反应 -> 事故/产量结算 -> 下一波
2. 按 20 分钟节奏推进：8 波，每波约 2.5 分钟（前兆期 45s + 决策锁定期 15s + 结算期 5s）

## Must Keep
- 核心机制必须保持：危险模式识别 + 信号次数限制 + 产量损失
- 核心循环必须保持：观察前兆 -> 发出预警或继续工作 -> 工人反应 -> 事故/产量结算 -> 下一波
- 20 分钟结构只作为节奏，不扩成大项目
- 3 个工作 Zone（A/B/C）作为操作目标，玩家拖拽预警旗到 Zone
- warning_tokens 每波重置，不可跨波积累

## Must Not Add
- 不做动作闪避；核心是预判与停工代价
- 不新增第二套主循环
- 不把小游戏扩成长期经营或开放世界

## Required State

| State | Init | Min | Max | 说明 |
|-------|------|-----|-----|------|
| warning_tokens | 3 | 0 | 3 | 每波重置的预警旗数量 |
| quota | 100 | 0 | 100 | 产量进度，降到 0 则产量结局 |
| injury_risk | 0 | 0 | 100 | 累计工伤风险，达到 100 则工伤结局 |
| false_alarm | 0 | 0 | 100 | 累计误报分数，达到 100 则信任结局 |
| wave | 1 | 1 | 8 | 当前波次 |

## State Thresholds & Endings

| 结局 | 触发条件 | 类型 |
|------|---------|------|
| 好结局 | wave=8 且 quota>=60 且 injury_risk<50 | 胜利 |
| 普通结局 | wave=8 且不满足好结局 | 普通 |
| 工伤结局 | injury_risk>=100（任意时刻）| 失败 |
| 信任结局 | false_alarm>=100（任意时刻）| 失败 |
| 产量结局 | quota<=0（任意时刻）| 失败 |

## 20-Minute Pacing

| 阶段 | Wave | 特征 |
|------|------|------|
| 教学 | 1-2 | 明显前兆，单 Zone 危险，假信号概率 10% |
| 压力 | 3-4 | 多 Zone 危险，假信号概率 25% |
| 混乱 | 5-6 | 模糊前兆，多 Zone 同时危险，假信号概率 40% |
| 高潮 | 7-8 | 极度模糊前兆，高惩罚，假信号概率 30% |

## Success
在限定时长内完成主循环（8 波），并稳定进入至少一个可结算结局

## Failure
关键状态崩溃（injury_risk/fale_alarm/quota 触发失败阈值），或在本轮主循环中被系统淘汰
