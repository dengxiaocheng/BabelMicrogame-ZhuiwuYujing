# Direction Lock: 坠物预警

## One Sentence
玩家识别塔上坠物前兆，决定何时让工人躲避而不耽误产量。

## Core Loop
1. 执行核心循环：观察前兆 -> 发出预警或继续工作 -> 工人反应 -> 事故/产量结算 -> 下一波
2. 按 20 分钟节奏推进：明显前兆 -> 多区域 -> 假信号 -> 模糊前兆和高惩罚

## Must Keep
- 核心机制必须保持：危险模式识别 + 信号次数限制 + 产量损失
- 核心循环必须保持：观察前兆 -> 发出预警或继续工作 -> 工人反应 -> 事故/产量结算 -> 下一波
- 20 分钟结构只作为节奏，不扩成大项目

## Must Not Add
- 不做动作闪避；核心是预判与停工代价
- 不新增第二套主循环
- 不把小游戏扩成长期经营或开放世界

## Required State
- resource
- pressure
- risk
- relation
- round

## Success
在限定时长内完成主循环，并稳定进入至少一个可结算结局

## Failure
关键状态崩溃，或在本轮主循环中被系统淘汰
